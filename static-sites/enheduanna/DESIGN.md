# Enheduanna design guide

- Last audited: 2026-07-19
Primary stylesheet: `static-sites/enheduanna/globals.css`

## Visual direction

当前网站采用具有强烈编辑感的“古代档案 + 现代展览图录”方向。它不是仿古卷轴，也不是通用历史博物馆模板，而是用纸张底色、深蓝、朱红、金色、粗线框、编号和巨大排版，把神庙制度、政治危机与作者声音组织成一场长篇展览。

页面氛围在不同章节中有明确变化：

- 世界、神庙和材料文化：浅纸色、蓝红几何结构。
- 流放与沉默：深蓝黑背景、金色和灰白文字。
- 伊南娜出现：金色、深蓝和红色形成神性与危险感。
- “我”的诞生：重新回到纸张和暖色，同时保留巨大剪影和引文。
- 沉默传统：深色尾声，许多手托举一个声音。

## Color system

| Token | Value | Use |
| --- | --- | --- |
| `--paper` | `#f4eedc` | 主纸张底色 |
| `--paper-deep` | `#e7dcc1` | 深一层纸张与分区 |
| `--blue` | `#153a8a` | 主结构色、导航、按钮和编号 |
| `--blue-dark` | `#0c2867` | 深色叙事章节 |
| `--red` | `#c83b2c` | 冲突、强调和交互焦点 |
| `--ink` | `#101010` | 正文、粗线和高对比结构 |
| `--muted` | `#665f52` | 辅助说明 |
| `--clay` | `#b88a56` | 泥土与古代材料的次要色 |
| 金色 | `#d8b668` 等 | 深色章节中的神性、文献和关键引文 |

不要把红、蓝、金改成柔和同色渐变；三者承担政治、制度和神性之间的对比。

## Typography

- 中文标题与长引文：宋体/思源宋体/Noto Serif CJK/SimSun fallback。
- 正文与 UI：苹方/思源黑体/Noto Sans CJK/微软雅黑 fallback。
- 英文档案编号和超大字：Arial Narrow、Roboto Condensed、DIN Condensed fallback。
- 大标题允许极大的字号、紧字距和断行，构成版面而不仅是文字内容。

## Layout principles

- 左侧固定 `52px` 蓝色 spine 保存 TS、项目名和编号 `004`。
- 顶部导航在桌面端给出章节入口，`details` 菜单提供完整目录。
- section 之间通过边框、背景色和巨量留白形成清晰章节切换。
- 采用不对称网格、档案卡、时间线、文物图文对照、引文带和巨型文字，而不是统一组件重复。
- 史料说明、图片 credit 和限定语必须贴近对应图像或论断。
- 页面故意允许个别章节信息密度很高；不要只因为“太长”就删除背景。

## Images

图片分为不同功能：

| 类型 | 例子 | 作用 |
| --- | --- | --- |
| 文物/历史材料 | 圆盘、乌尔军旗 | 提供物质证据和时代背景 |
| 创作者形象 | 线描肖像 | 建立人物入口，明确不是历史肖像 |
| 现代视觉文化 | Ishtar 动漫启发图 | 展示现代想象，必须标明非复原 |
| 概念插图 | `voices-lift.png` | 把“一个声音由许多人保存”可视化 |

图片说明是版面内容，不能只保留图片而删除来源或“非历史复原”标签。

## Interaction and motion

- 平滑锚点滚动。
- 导航下划线、按钮位移与硬阴影。
- 作品卡 hover 变色。
- `details` 展开完整目录和学术说明。
- 图片和按钮没有持续自动动画；交互主要用于浏览和强调。
- CSS 包含 `prefers-reduced-motion: reduce` 处理，新增动效必须继续尊重。

## Responsive behavior

- `1100px` 以下重新组织多列布局和导航。
- `760px` 以下进入移动布局：spine、topbar、hero、表格和多列内容必须改排，不能只整体缩小。
- 巨型标题需允许换行，避免当前桌面端的单行规则在窄屏造成横向滚动。
- 引文、来源列表和文物说明在移动端仍需保持完整可读。

## Established choices

- 网站强调编辑图录和展览感，不追求“古代羊皮纸”拟物真实性。
- 纸张纹理、硬边框、编号和大字号共同建立档案语言。
- 深色章节用于真正的叙事转折，不是随意的视觉轮换。
- 图片同时承担证据和现代再创作的不同角色，页面必须说清差异。
- 不是所有内容都放进圆角卡片；多数结构依赖直线、网格和连续版式。

## Needs creator confirmation

仓库没有完整保存此前所有 UI 尝试和否决记录。需要创作者确认：

- 是否有明确禁止的颜色、字体或布局版本。
- 左侧 spine 在未来改版中是否绝对保留。
- 现代动漫启发的 Ishtar 图是否有计划替换为其他视觉。

在确认前，不应把网站改成柔和博物馆模板、玻璃拟态或标准卡片落地页。
